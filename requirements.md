# 跳槽真实收益计算器需求说明文档

## 1. 项目背景

很多互联网从业者在跳槽或跨城市工作时，往往只关注薪资涨幅。例如从一个城市跳到另一个城市，薪资可能上涨 10%，但房租、通勤、餐饮、税费、搬家成本等也会同步增加。

如果生活成本上涨幅度超过薪资涨幅，用户的真实可支配收入反而可能下降。因此，本项目希望帮助用户判断：

> 跳槽后到底是真的赚更多，还是表面涨薪、实际降薪。

本产品不是一个普通薪资计算器，而是一个面向真实跳槽场景的决策辅助工具。

---

## 2. 产品目标

帮助用户在跳槽前快速比较当前工作和新 offer 的真实收益差异。

核心目标包括：

1. 计算当前城市的每月可支配收入。
2. 计算新城市 / 新 offer 下的每月可支配收入。
3. 对比跳槽前后的真实收入变化。
4. 估算一次性迁移成本的回本周期。
5. 给出是否值得跳槽的初步判断。

---

## 3. 目标用户

主要面向：

- 互联网从业者
- 应届生 / 实习生
- 准备跳槽的人
- 准备跨城市工作的人
- 同时拿到多个 offer、需要做选择的人

---

## 4. 核心使用场景

### 场景 1：跨城市跳槽

用户当前在 A 城市工作，拿到 B 城市的大厂 offer。  
虽然月薪上涨，但 B 城市房租和生活成本更高。  
用户希望判断跳槽后每个月到底能多存多少钱。

### 场景 2：表面涨薪但实际降薪

用户新 offer 月薪更高，但房租、通勤、生活费增加明显。  
系统计算后发现每月结余反而下降，并提醒用户存在实际降薪风险。

### 场景 3：计算搬家成本回本周期

用户跳槽需要支付搬家费、押金、中介费、临时住宿费等。  
系统帮助用户计算需要几个月才能覆盖这些一次性成本。

用户需要输入两组信息（支持设置默认值以降低输入门槛）：

每月工作天数：双休约 21.75天，大小周约 24天，单休约 26天

🏠 生存成本特征（月度支出）
每月房租：（未来可接入 DeepSeek API 自动估算）

每月生活费：（餐饮、水电煤网等）

每月交通费：（日常打车/地铁）

📦 跨城/跳槽特殊特征（仅针对新 Offer）
一次性迁移成本：（搬家费、违约金、租房中介费、押金沉没成本等）

一次性入职补贴：（签字费、安家费等，用于抵扣迁移成本）

5.2 计算模块
系统需要根据输入，在本地自动执行以下运算：

核心公式 1：月度生存结余
税后月薪 = 税前月薪 - 五险一金 - 个人所得税（按简易阶梯估算）

月度结余 = 税后月薪 - 每月房租 - 每月生活费 - 每月交通费

核心公式 2：年度真实总收益
年度结余 = (月度结余 × 12) + (税前月薪 × 溢出月数 × 税后折算率)

核心公式 3：真实时薪（反内卷指标）
每月总工时 = 每月工作天数 × 日均工作时长

真实时薪 = (年度结余 / 12) / 每月总工时

核心公式 4：迁移成本回本周期
净迁移成本 = 一次性迁移成本 - 一次性入职补贴

结余差额 = 新 Offer 月度结余 - 当前月度结余

回本周期（月） = 净迁移成本 / 结余差额
(注：如果结余差额为负数，提示“永远无法回本，实属倒贴打工”)
